# 평가 기준 상세 루브릭

---

## 섹션 1 · 시각적 위계 / 타이포그래피 / 컬러 대비

### 시각적 위계

**핵심 질문:** 처음 이 화면을 보는 사람이 3초 안에 "무엇을 해야 하는지" 알 수 있는가?

| 상태 | 기준 |
|---|---|
| 문제 없음 | 지배적 요소 1개. H1 → 서브 → CTA 시선 경로 명확. 액센트 ≤3개 요소 |
| 경미한 문제 | 주 포커스 식별 가능하나 경쟁 요소 존재 (동일 시각 무게의 CTA 2개 등) |
| 심각한 문제 | 지배적 요소 없음. 3개 이상 요소가 동시에 주목 요구. 배경이 전경 콘텐츠와 충돌 |

**시선 경로 테스트:** H1 → 서브타이틀 → CTA 순서로 시선이 자연스럽게 이동하는가? 그 경로를 방해하는 요소(플로팅 배너, 동일 크기 텍스트 블록)가 있는가?

### 타이포그래피

**체크리스트:**
- [ ] 폰트 패밀리 ≤2개 (웨이트 수는 무관)
- [ ] 인접한 타입 스케일 단계 간 비율 ≥1.2 (H1과 H2가 거의 같은 크기면 위계 붕괴)
- [ ] 본문 폰트 크기 ≥14px (이상적: 15–18px)
- [ ] 본문 line-height 1.4 이상 (이상적: 1.5–1.65)
- [ ] 본문 줄 길이 55–75ch
- [ ] 대형 디스플레이 텍스트에 자간(letter-spacing) 적용 여부
- [ ] 이탤릭, 볼드 등 강조 웨이트 사용 규칙이 일관적인가

**타입 스케일 참고 (Major Third 1.25×):**
```
12 → 15 → 19 → 24 → 30 → 37 → 48px
```

**타입 스케일 참고 (Perfect Fourth 1.333×):**
```
12 → 16 → 21 → 28 → 37 → 50px
```

### 컬러 대비

**WCAG AA 기준 (최소):**

| 텍스트 유형 | 최소 대비율 |
|---|---|
| 일반 본문 텍스트 (< 18px 또는 < 14px bold) | 4.5 : 1 |
| 대형 텍스트 (≥ 18px 또는 ≥ 14px bold) | 3 : 1 |
| UI 컴포넌트, 아이콘, 포커스 링 | 3 : 1 |

**WCAG AAA 기준 (권장):**

| 텍스트 유형 | 최소 대비율 |
|---|---|
| 일반 본문 텍스트 | 7 : 1 |
| 대형 텍스트 | 4.5 : 1 |

**액센트 컬러 포커스 규칙:**
- 1개 액센트 컬러
- 한 화면에서 3개 이하 요소에 적용
- 5개 이상 적용 시: 액센트가 아닌 기본 컬러로 기능 전락 → 포커스 신호 소멸

## 섹션 2 · 사용성 / 태스크 완료 직관성

### 태스크 완료 단계 분석

**핵심 질문:** 사용자가 이 화면에서 핵심 태스크를 완료하기까지 몇 번 클릭/탭이 필요한가? 각 단계가 다음 단계를 명확히 가리키는가?

**태스크 완료 방해 요소:**
- CTA가 스크롤 없이는 보이지 않는 위치
- 주요 액션과 보조 액션이 동일한 시각 무게
- 폼 제출 후 피드백 없음 (로딩 상태, 성공 상태 부재)
- 에러 메시지가 필드가 아닌 페이지 상단에만 표시

### 인터랙션 상태 체크리스트

모든 인터랙티브 컴포넌트에 아래 8개 상태가 존재해야 한다:

| 상태 | 확인 기준 |
|---|---|
| Default | 클릭 가능한 요소가 정적 요소와 시각적으로 구분되는가 |
| Hover | opacity: 0.9는 인지 불가. 컬러 변화, 언더라인, 배경, transform 등 명확한 변화 |
| `:focus-visible` | 키보드 포커스 링 존재. `outline: none` 단독 사용 금지. 링 대비 ≥3:1 |
| `:active` | 클릭 피드백. 미세한 transform 또는 컬러 변화 |
| Disabled | opacity 50% 단독이 아닌 시각적 구분 + `cursor: not-allowed` |
| Loading | 비동기 액션에 로딩 표시 (버튼 텍스트 변경 또는 스피너) |
| Error | 빨간 테두리만은 부족 — 에러 메시지 텍스트 + 아이콘 필수 |
| Success | 액션 완료 확인 피드백 (특히 폼 제출 후) |

### 모바일 체크리스트

- [ ] 탭 타겟 ≥44×44px
- [ ] 가로 스크롤 없음
- [ ] 핵심 CTA가 엄지 닿는 영역(화면 하단 1/3) 내 위치
- [ ] 텍스트 최소 14px (모바일 기준)
- [ ] 인풋 포커스 시 키보드가 CTA를 가리지 않는가

## 섹션 3 · 일관성 / 접근성

### 일관성 체크리스트

**컴포넌트 일관성:**
- [ ] 동일 역할의 버튼이 동일한 스타일 (padding, border-radius, 폰트 크기)
- [ ] 카드 컴포넌트 내부 패딩 일치
- [ ] 아이콘 크기와 스트로크 굵기 일치
- [ ] 동일 의미의 컬러가 다른 컨텍스트에서 다른 용도로 쓰이지 않음

**언어/카피 일관성:**
- [ ] 동일 액션에 동일한 레이블 ("저장", "저장하기", "변경사항 저장" 혼재 금지)
- [ ] 에러 메시지 어조 일관성

### 접근성 체크리스트

**구조:**
- [ ] 헤딩 순서: H1 → H2 → H3 (레벨 건너뜀 없음)
- [ ] 랜드마크 요소: `<nav>`, `<main>`, `<footer>` 사용
- [ ] 스킵 내비게이션 링크 (대형 앱)

**이미지와 아이콘:**
- [ ] 정보성 이미지: 내용을 설명하는 alt 텍스트 ("사람이 화이트보드 앞에 서서 발표하는 모습", "image" 또는 "" 금지)
- [ ] 장식용 이미지: `alt=""`
- [ ] 아이콘 전용 버튼: `aria-label` 또는 `aria-describedby`

**컬러와 의미:**
- [ ] 에러/성공/경고 상태가 컬러 + 텍스트 또는 아이콘으로 이중 표현
- [ ] 컬러만으로 정보를 전달하지 않음

**폼:**
- [ ] 레이블이 플레이스홀더와 분리되어 있음 (플레이스홀더 단독 레이블 금지)
- [ ] 에러 메시지가 해당 필드와 연결 (`aria-describedby`)
- [ ] 필수 필드 표시 방식 일관성

## 섹션 4 · 비즈니스 목표 달성 가능성

### 비즈니스 목표 식별

디자인을 보기 전에 이 화면의 핵심 목표를 추론한다. 목표가 불명확하면 비평 전에 질문한다.

**화면 유형별 주요 지표:**

| 화면 유형 | 핵심 지표 |
|---|---|
| 랜딩 페이지 | 회원가입/구매 전환율, 이탈률 |
| 온보딩 | 태스크 완료율, 드롭오프 단계 |
| 대시보드 | 핵심 데이터 인지 속도, 자주 사용하는 기능 도달 클릭 수 |
| 체크아웃 | 카트 이탈율, 폼 완료율 |
| 설정 화면 | 태스크 완료 시간, 지원 요청 감소 |

### 신뢰 신호 (Trust Signals) 체크

- [ ] 사회적 증명 (리뷰, 사용자 수, 로고)이 존재하는가?
- [ ] 보안 인디케이터 (SSL 뱃지, 결제 보안 아이콘)가 적절한 위치에 있는가?
- [ ] 위험 제거 요소 (무료 체험, 환불 보장)가 CTA 근처에 있는가?
- [ ] 숫자와 통계가 실제 데이터인가, 임의로 삽입된 것인가? (임의 수치는 신뢰를 오히려 해침)

### CTA 전환 마찰 분석

**마찰 요소:**
- CTA 텍스트가 모호함 ("제출", "확인" vs "무료로 시작하기", "지금 다운로드")
- CTA 주변에 불안을 유발하는 요소 (복잡한 약관, 과도한 필드 수)
- 가격 정보 없이 구매를 요구
- CTA까지 도달하기 위해 긴 스크롤 필요 (스티키 CTA 없음)

## 브랜드 시스템 이탈 판별 기준

### 이탈로 분류하는 것

- 하드코딩된 컬러 값이 토큰 값과 다름 (어떤 차이도 이탈로 표시 — 의도적인지 확인 요청)
- 시스템에 없는 폰트 사이즈 사용
- 스페이싱 스케일에 없는 임의 픽셀 값
- 공식 컴포넌트를 수정 없이 재사용해야 하는데 직접 구현됨
- 시스템에 정의된 컴포넌트와 다른 border-radius, shadow, border-width

### 이탈로 분류하지 않는 것

- 새로운 화면이라 시스템에 아직 패턴이 없는 경우 (이탈이 아닌 시스템 확장 후보로 표시)
- 실험적 변형임이 명확히 명시된 경우

### 보고 방식

이탈 항목은 사실로 보고한다. "틀렸다"가 아닌 "시스템과 다르다 — 의도적 변경인지 확인 필요"로 표현한다. 의도적인 이탈일 수 있고, 그 경우 시스템 업데이트가 필요하다.
