# Pitch Deck — Section Map

Canonical sections. Universal unless marked. Audience defaults below.

| #  | Section         | Purpose                                            | Investor | Sales |
|----|-----------------|----------------------------------------------------|----------|-------|
| 1  | Cover           | Brand mark, deck title, one-line value prop.       | ✓        | ✓     |
| 2  | Problem         | The pain. Sharp, specific, owned by the audience.  | ✓        | ✓     |
| 3  | Solution        | The product's core move. One sentence + visual.    | ✓        | ✓     |
| 4  | Market          | TAM/SAM/SOM (investor) OR market context (sales).  | ✓        | ◯     |
| 5  | Product         | Screens / demo / how-it-works.                     | ✓        | ✓     |
| 6  | Traction        | Revenue, users, growth curve, logos.               | ✓        | ◯     |
| 7  | Proof           | Case studies, testimonials, before/after.          | ◯        | ✓     |
| 8  | Business Model  | Pricing, unit economics, expansion path.           | ✓        | ◯     |
| 9  | GTM             | How customers find/buy. Channels, motion, CAC.     | ✓        | ✓     |
| 10 | Team            | Founders + key hires + why-us.                     | ✓        | ◯     |
| 11 | Roadmap         | Next 6–18 months — features, milestones.           | ✓        | ◯     |
| 12 | Ask / Next step | Investor: round size + use of funds. Sales: CTA.   | ✓        | ✓     |

✓ = default-on for that audience. ◯ = default-off but easy to add.

## Consolidation rules (small decks)

- Merge **Market + Business Model** for sub-12-slide investor decks.
- Merge **Product + Solution** when there's only one screen to show.
- Replace **Proof** with logo strip on Cover for very early sales decks.

## Order rules

- Investor: Cover → Problem → Solution → Market → Product → Traction → Business Model → GTM →
  Team → Roadmap → Ask.
- Sales: Cover → Problem → Solution → Product → Proof → GTM (light) → Ask.
- Never bury the Ask. It is always the last shipped section.
